The following is a list of noteworthy changes between ircd-ratbox 3.0
and ircd-ratbox-3.1.

Installation
------------
 o The name of the executable that is generated is now ircd-ratbox 
   instead of ircd.  Any crontabs should be updated to reflect this change.  
   Also you should remove any old executables in the $PREFIX/bin directory
   named ircd, the install process will not do this for you.
 o Nicklen is now a runtime configurable option(up to 30 characters)
   Nicklens beyond 30 will need a recompile as before.
 o Minimum nickname length is settable as well with serverinfo::nicklen_min
 

Config
------
 o Built in DNS RBL support via the blocklist {} configuration option
 o Add ipv6_tun_remap option. This remap transitional IPv6 technologies(teredo,
   6to4) to their IPv4 addresses for the purpose of channel bans and klines
 o Add options to configure TLS versions and ECDH support as well. 
 o WHOWAS history length can now be set at runtime with the whowas_length
   option
 o CERTFP support for connect {}, operator {} and auth{}.  CERTFP acts as an
   additional gate to these blocks and does not replace password/challenge.

Misc
----
 o Improve memory usage reporting
 o The code is much more reliant on C99 features, gcc 4.x and clang have
   been tested.  Let us know if another compiler does not compile it.
 o Fix time_t issues on 32bit OpenBSD with 64bit time_t
 o General code cleanups, dead code removal
 o REHASH memory leaks are gone!
 o Add ELIST new ELIST features C and T


==============================================================================
The following entries cover changes added in the 3.1.0-rc1 development
cycle on top of the items above.
==============================================================================

New oper commands
-----------------
 o TESTRBL <ip>: oper-only spot-check of a single IPv4 or IPv6 address
   against every configured blacklist{} zone. Reports per-zone HIT,
   MISS, CLEAN (NXDOMAIN), ERROR, or TIMEOUT without affecting any
   client and without modifying the issuing oper's connection state.
   /HELP TESTRBL describes the output.

New STATS letters
-----------------
 o STATS n / STATS N (oper-only): per-zone DNSBL counters. One line per
   configured blacklist{} of the form
       n <zone> queries=N matches=N misses=N
   /HELP STATS now lists the letter.

Config
------
 o blocklist { } accepted as an alias for blacklist { } in ircd.conf
 o map_oper_only=no combined with serverhide::flatten_links=yes now
   emits a config-load warning, since flatten_links overrides the
   intended /MAP visibility for non-opers.

DNSBL hardening
---------------
 o rblquery_t.queryid widened from uint16_t to uint32_t. The DNS
   helper IDs are 32-bit; truncation caused queryid mismatches and
   use-after-free on disconnect after roughly 65k lookups.
 o IPv6 client lookups: rbl_isv6() returns the bit flag (0x2), so the
   previous "!= true" comparison made every IPv6 client skip every
   blacklist. Replaced with a proper truthiness check.
 o rbl->refcount is now bumped before lookup_hostname() to handle
   synchronous failed_resolver callbacks safely.
 o rbl_destroy()/rbl_clear_rbllists() ordering corrected to avoid a
   stale rb_dlinkDelete on previously-removed list entries during a
   REHASH while DNS lookups are in flight.

TLS
---
 o TLS 1.3 NewSessionTicket suppression on server contexts. The
   existing SSL_OP_NO_TICKET flag only covered pre-1.3 stateless
   tickets; SSL_CTX_set_num_tickets(0) is now also called when built
   against OpenSSL 1.1.1 or newer (skipped for LibreSSL). Works around
   stalls observed against ircds linked to early OpenSSL 1.1.1
   patchlevels with known TLS-1.3 NST/post-handshake races.
 o Raw ERR_error_string format string is no longer leaked into the
   user-facing SSL quit message.

Resolver / hostmask / bandb
---------------------------
 o resolver/res.c proc_answer() now bounds-checks rd_length against
   the end of the packet buffer, fixing a 4- to 16-byte over-read on
   crafted T_A/T_AAAA/T_CNAME responses.
 o parse_netmask() in src/hostmask.c clamps negative CIDR bit counts
   to zero, fixing a UB shift count when /-N appeared in DLINE/KLINE/
   TESTLINE input or in the config file.
 o serverinfo::nicklen now clamps to NICKLEN-1 on overflow and to a
   sensible minimum on underflow instead of writing the raw configured
   value into ServerInfo.nicklen, where it could overflow stack/heap
   nick buffers in mr_nick / m_nick / set_initial_nick.
 o bandb_handle_ban() validates parc before dereferencing parv[]
   entries; bandb_helper is null-checked in bandb_add/bandb_del so
   that a failed bandb start no longer crashes the server on the
   first ban update.

Net / I/O
---------
 o linebuf chop-trailing loop guarded against size_t underflow on
   pathological inputs.
 o failed_resolver callback is now deferred to the next event loop
   iteration to avoid use-after-free across the dns / s_auth boundary.
 o AWAY message updates (changing an existing away string, not just
   toggling on/off) are now propagated to peer servers; the previous
   code only sent the on/off transition.
 o Accept callback no longer fires on a closed fd after rb_set_nb()
   failure on the newly-accepted socket.

Rehash / cache
--------------
 o call_hook() iterates with RB_DLINK_FOREACH_SAFE so a hook callback
   that removes itself does not corrupt the iteration.
 o motd_path is preserved across REHASH when the new config omits it,
   matching the long-standing behaviour of other path settings.
 o free_cachefile() no longer leaks the dlink_node for empty MOTD/help
   lines.

Misc correctness
----------------
 o scache_remove() is now called when a server departs, fixing an
   unbounded heap growth on burst/split-rejoin storms.
 o YESNO command-line option handling no longer writes through a
   wrong-typed pointer and clobbers neighbouring globals.
 o ipv6_tun_remap matching restored after a regression that disabled
   ban matching against the remapped IPv4 address; garbage rewrite of
   the underlying IPv4 sockaddr was also removed.
